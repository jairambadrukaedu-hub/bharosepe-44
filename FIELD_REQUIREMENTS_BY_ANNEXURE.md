## UNIFIED DATABASE ARCHITECTURE

**Single `form_submissions` table with:**
- **Common fields:** ~15 columns (all transactions)
- **Goods columns:** ~200 columns (electronics, vehicles, fashion, jewelry, etc.)
- **Service columns:** ~518 columns (software, design, content, photography, coaching, repair, cleaning, marketing, consulting)
- **Total:** ~730+ columns (within PostgreSQL 1,600 limit)

**Master contract generation:**
```
IF transaction_type = 'goods' THEN use master_contract_goods.docx
ELSE IF transaction_type = 'services_software' THEN use master_contract_software.docx
ELSE IF transaction_type = 'services_consulting' THEN use master_contract_consulting.docx
... (etc for each service type)
```

**Key Benefits:**
✅ Single contract generation query (fast)
✅ One data source for all transaction types
✅ Scalable to 15+ service types
✅ Well within database limits
✅ Future-proof architecture
